Executive Summary Bloomberg Terminal costs $24,000 per year. It is the single most used tool in professional finance. The reason is not its data — it is its interconnectedness. Click any entity anywhere and everything else on screen reorganizes around it. Traders pay $2,000/month just for that traversal. No equivalent exists for crypto-native users. Existing crypto dashboards are siloed: one tab for news, another for prices, another for ETF flows, another for macro calendars. Every time you want to connect two pieces of information you open a new tab. CryptoDesk is the first crypto terminal where everything is connected. Built entirely on SoSoValue&apos;s API — the only data provider that combines ETF flows, macro events, SSI indices, BTC treasuries, crypto stocks, and news in a single stack — CryptoDesk lets any user start anywhere and reach any related data in two clicks, then act on it directly via SoDEX. The one-sentence pitch CryptoDesk is what you get when you give Bloomberg&apos;s interconnectedness model to a crypto-native data stack — built on SoSoValue, executed on SoDEX. 2. The Problem — In Plain Language 2.1 What is the current user experience? Meet a typical crypto investor. They want to understand one thing: &apos;CPI drops tomorrow — should I adjust my BTC position?&apos; Here is their current workflow: Open a macro calendar site to find the CPI date and forecast number. Open CoinDesk or SoSoValue to find ETF flow data — read 2–3 articles to get a number. Open TradingView for BTC price chart. Open Twitter to read analysts — gets 15 contradictory opinions. Try to remember what happened last time CPI dropped. Give up. Hold. Gets surprised by the move. The information to make this decision exists. It is scattered across 5+ tools. The user does not have time to synthesize it. So they act on incomplete information — or do not act at all. 2.2 What is missing? Three things that Bloomberg has and crypto tools do not: Connected data — click a news headline and see the related price chart and ETF flow automatically. Context traversal — start anywhere (news, price, event) and reach any related data in two clicks. Integrated execution — see the signal and act on it in the same screen, without switching tools. 3. The Solution — CryptoDesk 3.1 Core concept A single-page terminal with a universal search bar and three persistent panels. Search any entity — a crypto asset, a macro event, a stock, an index — and all three panels instantly reorganize to show connected data about that entity. 3.2 The three-panel layout Panel Name What it shows Panel A (Left) Signal Feed News, events, and alerts related to the searched entity. Real-time, filterable by category. Panel B (Center) Chart &amp; Context Price chart or performance chart for the entity, with macro event overlays, ETF flow bars, and index constituent data layered on top. Panel C (Right) Action Layer Live SoDEX price, scenario brief (AI-generated), trade setup suggestion, and execution button. SoDEX is load-bearing here. 3.3 The four search contexts (Wave 1 scope) Every search term resolves into one of four contexts. Each context pulls different SoSoValue endpoints but renders in the same three-panel layout. This is the Bloomberg insight: one interface, infinite contexts. Technical Architecture 4.1 System overview CryptoDesk has two layers: a thin Node.js backend that proxies and caches SoSoValue API calls, and a React frontend that renders the three-panel terminal UI. The backend exists for one reason: the SoSoValue rate limit is 20 requests/minute. Without a caching proxy, the live demo breaks within 2 minutes. 4.2 Backend — Node.js / Express Route Upstream Call Cache TTL /api/news GET /news + /news/hot + /news/search 2 minutes /api/macro GET /macro/events + /{event}/history 1 hour (historical), 5 min (calendar) /api/etf GET /etfs/summary-history + /market-snapshot 5 minutes /api/currency/:id GET /currencies/{id}/market-snapshot + /klines 30 seconds /api/index/:ticker GET /indices/{ticker}/constituents + /klines + /market-snapshot 1 minute /api/stock/:ticker GET /crypto-stocks/{ticker}/market-snapshot + /klines + /btc-treasuries 5 minutes /api/ai/brief OpenRouter (Mistral 7B or DeepSeek) 10 minutes per prompt hash /api/sodex/ticker SoDEX GET /markets/tickers (public, no auth) 5 seconds Cache strategy: in-memory Map keyed by route + params. TTL checked on every request. On cache miss, upstream call fires, response stored. No Redis needed for Wave 1 demo. 4.3 Frontend — React + Vite + TypeScript State management: Zustand (lightweight, no Redux boilerplate) Charts: Recharts — ComposedChart for overlay data (price + ETF bars + event markers) Styling: Tailwind CSS — dark theme, monospace font for terminal aesthetic Search: debounced input (300ms), resolves to one of 4 context types via keyword mapping WebSocket: SoDEX live price stream for Panel C (wss://mainnet-gw.sodex.dev/ws/spot) Deployment: Vercel (frontend) + Railway free tier (backend) 4.4 Search resolution logic The search bar is the heart of CryptoDesk. When a user types, the frontend resolves the input to a context type: if input matches known SSI ticker (ssimag7, ssilayer1...) → Context 4 (Index) if input matches crypto stock ticker (MSTR, COIN, MARA...) → Context 3 (Stock) if input matches macro keyword (CPI, FOMC, NFP...) → Context 2 (Macro) else → Context 1 (Currency) — call /currencies?symbol={input} 4.5 External APIs used SoSoValue API Primary data source — news, ETF, macro, indices, currencies, BTC treasuries, crypto stocks. All proxied through backend. SoDEX REST + WebSocket Live price ticker (public, no auth). Order placement in Wave 2 only. Binance Public API 1-hour klines for macro reaction analysis (Context 2 only). Free, no auth required. OpenRouter AI narrative generation via Mistral-7B-Instruct or DeepSeek-chat. Free tier. Used only for Panel C briefs. 5.
